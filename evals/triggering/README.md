# Triggering Evals

Executable checks for the six frontend-first v3 repo-understanding skill descriptions:

- representative orchestration and leaf-worker requests route to one unique skill;
- every skill advertises v3 metadata and its public trigger phrases;
- product-intent, runtime-external, and deterministic diagnostic requests never route directly to a leaf agent;
- SKILL.md prose stays runtime-neutral.

Run with `npm run eval:triggering`.
